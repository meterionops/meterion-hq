# Pertti — North Star

## Purpose

Pertti is being built as a **governance-driven supervisory AI operating system**.

Its purpose is to:
- supervise multiple projects simultaneously
- maintain structured memory of operations, decisions, and outcomes
- evaluate alternative actions before execution
- guide work through planning, routing, and task assignment
- enforce governance and capability constraints
- learn from outcomes in a controlled, non-adaptive contract layer
- continuously identify new operational and business opportunities

Pertti is not:
- a chatbot
- a single autonomous agent
- an execution engine

Pertti is:
> a portfolio-level supervisory intelligence layer

---

## Core Role

Pertti operates as a **portfolio supervisor** across:

- CityOS (multi-city operational systems)
- new venture projects
- discovery pipelines
- country-level operational teams
- other Meterion projects

It provides:
- situational awareness
- structured reasoning
- planning and coordination
- governance enforcement
- cross-project insight

---

## End-State Capabilities

### 1. Memory

Pertti maintains structured memory of:
- findings
- decisions
- proposals
- outcomes
- feedback
- project state
- portfolio trends

---

### 2. Reasoning

Pertti evaluates:
- current system state
- possible actions
- expected consequences
- risks and constraints
- uncertainties

This is implemented through the World Model and related layers.

---

### 3. Governance

Pertti operates strictly under:
- capability constraints
- governance rules
- project boundaries
- human approval requirements

It never executes actions outside approved boundaries.

---

### 4. Planning

Pertti converts reasoning into structured plans:
- goals
- steps
- dependencies
- blockers
- handoffs

Planning is separate from execution.

---

### 5. Routing and Coordination

Pertti determines:
- what should be done
- where it should be done
- which role or system should handle it
- which adapter (OpenClaw, Codex, human, API) is appropriate

It does not execute, only routes.

---

### 6. Audit and Learning

Pertti tracks:
- what was proposed
- what was approved
- what was routed
- what happened
- what worked and what failed

Learning is:
- structured
- explicit
- non-autonomous (no self-modifying behavior)

---

### 7. Opportunity Discovery

Pertti identifies opportunities from:

#### Existing operations
- coverage gaps
- performance patterns
- repeated failures or bottlenecks
- high-performing structures
- scalable workflows

#### New opportunities
- adjacent business models
- new markets (city/country expansion)
- new operational templates
- new venture hypotheses

---

## Architectural Model

Pertti is structured as layered supervisory architecture:

### Layer 1 — Kernel
- identity
- governance
- project registry
- execution context

### Layer 2 — Supervisory Reasoning
- world model
- recommendation boundary
- decision policy

### Layer 3 — Operational Coordination
- execution routing
- project adapter resolution
- role task model
- task dispatch boundary

### Layer 4 — Memory and Audit
- outcome ledger
- learning / feedback
- supervisory run sessions

### Layer 5 — Planning and Simulation
- planner
- consequence analysis
- scenario evaluation
- simulation (future)

### Layer 6 — Opportunity Intelligence (future)
- gap detection
- pattern recognition
- source intelligence
- venture discovery

### Layer 7 — Execution Ecosystem
- OpenClaw
- Codex
- human operators
- APIs and external systems

Pertti itself does not execute actions.
It supervises execution through controlled interfaces.

---

## CityOS Context

In the context of CityOS, Pertti enables:

- management of hundreds of city-level systems
- comparison across cities and countries
- identification of coverage gaps
- prioritization of expansion and fixes
- simulation of rollout strategies
- structured coordination of operational work

---

## Multi-Project Supervision

Pertti is designed to:

- operate across multiple projects simultaneously
- maintain strict project isolation
- reuse supervisory logic across projects
- support country-level AI team structures
- coordinate shared capabilities across projects

---

## Guiding Principles

- **Propose-first, not execute-first**
- **Governance over autonomy**
- **Separation of reasoning and execution**
- **Deterministic contract layers**
- **Explicit auditability**
- **Human override is absolute**
- **No hidden behavior**

---

## Long-Term Vision

Pertti evolves into a system that can:

- supervise large-scale operational systems (e.g. CityOS)
- coordinate multi-country AI operations
- simulate strategic decisions before execution
- continuously improve operations based on structured feedback
- identify and prioritize new venture opportunities

---

## Summary

Pertti is not an agent.

Pertti is:
> a supervisory operating system for portfolio management, operational intelligence, and venture discovery.
