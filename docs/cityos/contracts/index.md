# Contracts Index

## Purpose

This index lists system contracts that define how different parts of CityOS interact.

Contracts define expectations between components, not internal implementation.

---

## What belongs here

Contracts should define:

- API contracts (inputs / outputs)
- internal service boundaries
- integration expectations
- schema-level guarantees
- request/response shapes

## What does NOT belong here

Do not place these here:

- internal data model definitions → use `models`
- processing logic → use `specs`
- operational procedures → use `playbooks`
- onboarding / documentation rules → use `onboarding`
