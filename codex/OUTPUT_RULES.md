# Codex Output Rules (Pertti)

All Codex outputs MUST follow this format:

## Allowed output
- TypeScript code only

## Forbidden output
- explanations
- reasoning
- markdown
- comments (unless part of existing code style)
- multiple files unless explicitly requested

## Style rules
- minimal implementation
- no overengineering
- no abstraction without clear need
- match existing file style exactly

## Golden rule
If the output contains anything other than code, it is invalid.
